# EasyEvent 官网设计说明

## 参考原则

本阶段参考 Aaru 一类前沿科技产品官网的高级感，学习的是设计原则，而不是具体资产：

- 极简导航。
- 大标题与短句叙事。
- 强留白和高对比。
- 细线框与编号结构。
- 抽象生成式视觉。
- 少量高饱和强调色。

## 如何避免复制

EasyEvent 没有复制参考网站的：

- 文案。
- 图片。
- 视频。
- Logo。
- 品牌符号。
- 人物视觉。
- 具体模块布局。
- 商业素材。

视觉内容全部使用 CSS、SVG 和站内文案生成，生成式赛事视觉围绕 EasyEvent 自己的场景：赛事信号、点位网络、移动队伍、运营系统。

## EasyEvent 自己的视觉语言

关键词：

- Editorial Futurism
- Research Lab
- Event Operating System
- Monochrome + Acid Green
- Thin Borders
- Oversized Typography
- Generative Event Graph
- Calm but Powerful

色彩：

- paper：`#f5f3ee`
- ink：`#060606`
- muted：`#66615a`
- acid：`#00f06a`
- dark：`#030303`

排版：

- 大标题使用系统 serif。
- 正文和导航使用系统 sans。
- 编号、参数和标签使用 monospace。

## 页面结构

1. HeroSection：生成式赛事图 + 巨型标题 + 酸绿判断卡。
2. ThesisSection：说明赛事不该依赖表格和群聊。
3. ProductSystemSection：六个赛事操作系统能力。
4. TemplateSection：模板矩阵与 EventConfig 概念。
5. LiveSignalsSection：健康 / 可穿戴信号路线和隐私边界。
6. AiAssistantSection：AI 助手路线、prompt panel 和人工确认边界。
7. DownloadSection：Web / Android / iOS / Windows 发布入口。
8. ContactSection：early access brief 和联系表单。
9. Footer：极简品牌收尾。

## 截图建议

- Hero 首屏：展示品牌调性和生成式赛事视觉。
- Thesis + Product System：展示 EasyEvent 不只是报名表。
- Template matrix：展示多赛事复用能力。
- Live Signals：展示健康 / 可穿戴未来路线和隐私边界。
- AI Assistant：展示未来 AI layer，但强调人工确认。
- Download + Contact：展示真实产品发布和早期访问入口。
