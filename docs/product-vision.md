# EasyEvent 产品愿景

## 新定位

EasyEvent 的目标是成为中小型体育赛事操作系统，而不是单场赛事工具。它面向城市定向赛、校园跑步赛、篮球赛、亲子运动会、企业团建赛、徒步 / 越野活动等不同场景，提供可复用的赛事配置、报名审核、签到检录、现场执行、通知确认和数据复盘能力。

英文定位：

EasyEvent is a reusable operating system for sports event operations.

## 不只是鲁滨逊

珠海城市鲁宾逊趣味定向赛只是第一个代表性场景。EasyEvent 的核心抽象是 `EventConfig`：

- 报名模式
- 项目 / 路线
- 队伍人数规则
- 成员字段
- 材料要求
- 签到方式
- 点位任务
- 通知类型
- 数据复盘指标

通过配置而不是写死页面，EasyEvent 可以复制到不同赛事格式。

## 官网定位

官网承担三个任务：

1. 介绍 EasyEvent 的产品定位。
2. 展示可复用的赛事运营能力。
3. 提供下载入口和联系入口。

官网不承载业务后台，也不接支付、健康 API、AI API 或地图服务。

官网视觉方向采用 editorial futurism：高对比黑白 / 米白空间、超大 serif 标题、细线结构、编号模块、酸性绿色点睛和抽象生成式赛事视觉。它强调 EasyEvent 是“赛事操作系统”，不是单个模板页面。

## 未来集成路线

未来可探索的集成方向：

- Apple Health / HealthKit
- Android Health Connect
- Apple Watch / Wear OS
- Fitbit / Garmin / Samsung Health
- GPS / route services
- AI Assistant
- Windows Desktop

这些能力应逐步进入产品路线，而不是一次性全部接入。

## AI 助手边界

AI Assistant 的优先方向是只读分析和草稿生成：

- 生成赛事模板草稿。
- 起草通知公告。
- 解释报名缺失项。
- 汇总签到和现场状态。
- 找出需要关注的队伍。
- 生成赛后报告草稿。

关键操作必须人工确认。AI 不应在未经确认的情况下自动审核、驳回、修改报名记录或发布重要通知。

AI 助手不提供医疗建议，不替代赛事工作人员判断。

## 健康数据边界

健康和可穿戴数据必须遵守以下边界：

- 用户明确授权。
- 最小化读取，只取赛事运营所需字段。
- 不做医疗诊断。
- 不把个人心率等敏感数据公开给队友。
- 管理端只展示必要状态提示，例如活动异常、长时间未动、需关注。
- 用户应能理解数据用途，并能撤回授权。

## 产品原则

- 先完成赛事运营闭环，再扩展智能能力。
- 先做配置化复用，再做行业模板库。
- 先做可解释状态流，再做自动化建议。
- 先保护隐私，再追求实时性。
