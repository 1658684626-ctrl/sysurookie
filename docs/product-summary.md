# EasyEvent 产品说明

## 产品定位

EasyEvent 易赛通是一个面向中小型体育赛事的一站式数字化管理系统课程 MVP。

## 痛点分析

- 报名、审核、签到、通知和复盘分散在不同工具中。
- 赛事规则复杂，人工核对容易漏审、错审。
- 现场执行状态不可视，队伍进度和异常情况依赖人工沟通。
- 赛后数据难以沉淀，下一届赛事优化缺少依据。

## 方案设计

EasyEvent 用 EventConfig 描述赛事模板，用 mock data 和 localStorage 串起报名、审核、签到、点位、通知和复盘。首个演示案例是珠海城市鲁宾逊趣味定向赛，同时支持校园跑步赛、篮球赛等模板。

## 模块说明

- 赛事模板配置：配置报名模式、项目路线、成员字段、材料规则、点位和通知类型。
- 参赛者端：报名组队、资料填写、查看状态、点位打卡、通知确认。
- 管理端：审核资料、签到检录、现场执行管理、通知公告、数据复盘。
- 数据复盘：汇总报名状态、签到率、完赛率、异常关注、点位任务和通知确认率。

## 创新点

- 不为单场赛事写死，通过 EventConfig 支持多赛事复用。
- 不止做报名表，而是覆盖赛前、赛中、赛后运营闭环。
- 用状态流降低参赛者和管理端的信息不对称。
- 用复盘指标沉淀赛事运营经验。

## 技术可行性

当前 MVP 使用 Vite、React、TypeScript、Tailwind CSS、mock data 和 localStorage。真实上线可以扩展到小程序端、云数据库、文件上传、二维码签到、地图导航和消息推送。

## MVP 边界

本项目不实现真实后端、数据库、登录、支付、短信、微信、GPS、地图、文件上传、复杂权限和商业级 BI。
