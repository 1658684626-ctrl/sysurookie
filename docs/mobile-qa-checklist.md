# EasyEvent 移动端 QA 检查清单

检查建议宽度：

- 手机：390px
- 平板：768px
- 桌面：1280px

每个页面统一检查：

- 390px 手机宽度是否可用。
- 768px 平板宽度是否可用。
- 页面整体是否出现横向溢出。
- 关键按钮是否容易点击。
- 表格是否可以横向滚动。
- 长文本、长队伍名、长通知内容是否换行。
- 空状态是否有中文提示。
- 页面标题是否能说明当前模块用途。
- 底部是否有足够留白。
- Capacitor App 中是否可能被状态栏或 Home Indicator 遮挡。

## 页面清单

### 1. 演示模式页

- 检查产品定位、痛点、闭环和演示流程是否一屏内易读。
- 检查“一键生成完整演示案例”和复制讲稿按钮是否可点击。
- 检查讲稿 fallback 文本框是否不会撑破页面。

### 2. 项目概览页

- 检查赛事模板选择器是否可操作。
- 检查核心指标卡片在手机端是否单列或合理换行。
- 检查点位概览长名称是否换行。

### 3. 参赛者端首页

- 检查赛事卡片是否清晰。
- 检查步骤导航在手机端横向滚动是否顺滑。
- 检查“我的报名记录”卡片是否可点击。

### 4. 参赛者端赛事详情页

- 检查项目 / 路线卡片是否从 EventConfig 动态展示。
- 检查材料规则和人数规则长文本是否换行。

### 5. 参赛者端报名 / 成员资料页

- 检查表单字段在手机端是否单列。
- 检查添加、编辑、删除成员按钮是否容易点击。
- 检查材料文件名输入框不会溢出。

### 6. 参赛者端报名状态页

- 检查状态标签和缺失项提示是否清楚。
- 检查驳回原因、审核通过、报名成功文案是否不混淆。

### 7. 参赛者端签到码卡片

- 检查检录码是否大号显示并可换行。
- 检查未报名成功时不会展示正式签到码。
- 检查已签到 / 已出发状态文案是否清楚。

### 8. 参赛者端点位打卡页

- 检查点位列表在手机端是否像任务卡。
- 检查“模拟到达点位”和“提交任务”按钮不被底部遮挡。
- 检查备注、文件名等输入内容是否换行。

### 9. 参赛者端通知公告页

- 检查紧急通知视觉是否足够醒目但不过度。
- 检查“我已知悉”按钮可点击。
- 检查已确认时间显示是否换行。

### 10. 管理端首页

- 检查二级模块导航是否可以横向滚动。
- 检查当前赛事选择器和重置按钮在手机端是否堆叠合理。

### 11. 管理端审核工作台

- 检查报名列表表格是否横向滚动。
- 检查详情面板在手机端是否排在列表下方。
- 检查驳回原因为空时提示中文清楚。

### 12. 管理端签到检录页

- 检查检录码输入框和“检录签到”按钮是否适合现场触屏。
- 检查可检录名单是否横向滚动。
- 检查错误检录码提示是否说明下一步。

### 13. 管理端现场执行页

- 检查执行列表是否横向滚动。
- 检查点位任务待审核、异常关注和完赛按钮是否清楚。
- 检查异常原因输入为空时提示中文清楚。

### 14. 管理端通知公告页

- 检查通知列表在手机端是否可横向滚动。
- 检查通知编辑表单单列展示。
- 检查标题为空、内容为空、目标为空时提示中文清楚。

### 15. 管理端数据复盘页

- 检查核心指标卡片适合截图。
- 检查漏斗、项目表现、点位复盘、通知复盘表格横向滚动。
- 检查“复制复盘摘要”按钮和 fallback 文本区域可用。

## 本阶段已做的基础处理

- AppShell 增加顶部和底部 safe area padding。
- 顶部主导航使用 safe area sticky top。
- 管理端二级导航支持手机横向滚动。
- 多个管理端表格改为横向滚动容器。
- 通用输入、按钮和卡片改为更适合触屏的尺寸和圆角。
